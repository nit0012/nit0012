### ⚡ [CODTRAKR](https://github.com/Nitishojha00/COD_TRAKR)
**Competitive Programming Tracker** &nbsp;·&nbsp; `Sep 2024 – Present`

Production-grade platform tracking progress across LeetCode, Codeforces, CodeChef & GFG with multi-layer caching and background job queues.

**🔑 Highlights**
- 📦 **3-layer cache system** — Redis (15 min TTL) + dashboard cache (24h TTL) + client-side localStorage → **60–80% cache hit rate**
- ⚙️ **BullMQ queue** (concurrency=5) — Redis-locked background worker refreshes stale data silently, server never blocks on Puppeteer scrapers
- 🔍 **4 custom scrapers** — GraphQL + REST for LeetCode/CF; Puppeteer headless for CodeChef/GFG; per-platform Redis cache + soft-delete (`isActive=false`) preserving full history
- 🔐 **OTP login** via AWS SES + stateless JWT (HTTP-only cookies, bcrypt) + rate limiting from scratch using Redis `INCR+EXPIRE` — zero external library

### 💬 [Talksy](https://github.com/Nitishojha00/Talksy)
**Real-Time Chat Application** &nbsp;·&nbsp; `Oct 2024 – Present`

Fault-tolerant, horizontally scalable MERN chat app with WebSocket support and rich messaging features.

**🔑 Highlights**
- 🔌 **Fault-tolerant WebSocket** — ping-pong heartbeat (25s interval, 60s timeout) + automatic long-polling fallback → **99.9% connection uptime**, zombie connections auto-pruned
- 🔐 **Stateless JWT** (48h expiry, bcrypt) + strict CORS whitelisting — secures both REST APIs and Socket.IO connections
- 💬 **Feature-rich chat** — 1-on-1 & group rooms, typing indicators, online status, full CRUD; dynamic room allocation
- ⚡ **MongoDB aggregation** — optimized population pipelines reduce DB round trips fetching nested chat + user + message data
